# Support

## Getting Help

For setup, usage, or contribution questions:

- start with [README.md](README.md)
- check [docs/arch_skill_usage_guide.md](docs/arch_skill_usage_guide.md)
- open a GitHub issue with the command you ran, what happened, and what you expected

## Bug Reports

When reporting a bug, include:

- the skill or install target involved
- your host runtime, such as Codex CLI, Claude Code, or Gemini CLI
- the command output or error message
- whether the issue happens before or after `make install`
- any relevant local verification output, such as `npx skills check` or `make verify_install`

Do not include credentials, tokens, private repository contents, or personal machine details that are not necessary to reproduce the issue.

## Security Issues

Report vulnerabilities through [SECURITY.md](SECURITY.md). Do not file public issues for security-sensitive reports.
